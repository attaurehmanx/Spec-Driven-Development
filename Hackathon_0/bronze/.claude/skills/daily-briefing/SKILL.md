# Daily Briefing Skill

## Description
Generates daily briefings and summaries for the AI Employee system. Reviews completed tasks, pending items, and provides actionable insights.

## Trigger Phrases
- "generate daily briefing"
- "create morning briefing"
- "daily summary"
- "what happened today"
- "daily report"

## Instructions

You are the Daily Briefing Generator for the AI Employee system. Your role is to create comprehensive daily summaries.

### 1. Data Collection
Gather information from:
- `vault/Done` folder - completed tasks from the last 24 hours
- `vault/Needs_Action` folder - pending tasks
- `vault/Pending_Approval` folder - items awaiting approval
- `vault/Dashboard.md` - current system state
- `vault/Logs` folder - recent activity logs

### 2. Briefing Structure
Create a briefing in `vault/Briefings/YYYY-MM-DD_Daily_Briefing.md` with:

```markdown
---
generated: [timestamp]
period: [date range]
type: daily_briefing
---

# Daily Briefing - [Date]

## Executive Summary
[2-3 sentence overview of the day]

## Completed Tasks
- [x] Task 1 - [brief description]
- [x] Task 2 - [brief description]

**Total Completed**: X tasks

## Pending Tasks
- [ ] Task 1 - Priority: [High/Medium/Low]
- [ ] Task 2 - Priority: [High/Medium/Low]

**Total Pending**: Y tasks

## Awaiting Approval
- Item 1 - [description]
- Item 2 - [description]

**Total Awaiting**: Z items

## Alerts & Issues
[Any problems, errors, or items requiring attention]

## Recommendations
1. [Actionable recommendation]
2. [Actionable recommendation]

## Tomorrow's Focus
- Priority 1
- Priority 2
- Priority 3

---
*Generated by AI Employee*
```

### 3. Analysis Guidelines
- Identify patterns in completed tasks
- Flag overdue items
- Highlight urgent pending tasks
- Note any system errors or issues
- Provide actionable recommendations

### 4. Timing
- Generate briefings at 8:00 AM (morning briefing)
- Can be triggered manually anytime
- Store in `vault/Briefings` folder

### 5. Update Dashboard
After generating briefing:
- Update vault/Dashboard.md with briefing summary
- Update "Last Check" timestamp
- Update task counts

## Example Usage

**User**: "Generate daily briefing"

**Response**:
1. Collect data from all relevant folders in vault/
2. Analyze completed and pending tasks
3. Create briefing file in vault/Briefings/
4. Update vault/Dashboard.md
5. Report summary to user

## Output Format
Provide a concise summary:
- Briefing created: [filename]
- Completed tasks: X
- Pending tasks: Y
- Awaiting approval: Z
- Key recommendations: [list]
